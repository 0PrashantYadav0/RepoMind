About the Project

RepoMindAbout
The problem

Your repo knows.
You can't ask it.

"Why was auth built this way, and who decided it?" The answer is buried across commits, closed PRs and stale issues — and keyword search can't connect the dots.

RepoMind turns commits, PRs, issues, people & chat into one typed knowledge graph, then answers by multi-hop traversal — so every answer comes with the path that proves it.

Tech Stack & Architecture

RepoMindTech
The stack — deliberately lean

Built on boring, solid tools

  • Python 3.11 · FastAPI · Uvicorn — API + webhook server
  • Pydantic · GitPython · PyGithub — typed models & sources
  • Cognee — hybrid graph + vector memory (local or cloud)
  • Vanilla JS + force-graph — GitHub-themed web UI
  • pytest · ruff · uv — 109 tests, all offline
RepoMindTech · Stack
Architecture

One idempotent pipeline, three layers

Sources

Git, GitHub, Slack, Discord — all behind one Source interface.

Ingestor

One path: upsert · resolve refs · park/retry · tombstone. Every trigger funnels through it.

Graph Store

Exact typed KB kept local + Cognee hybrid recall (memory / local / cloud).

File→ Commit→ PullRequest→ Reviewer→ Issue
RepoMindTech · Architecture
Correctness-first by design

Safe to re-run, always

Idempotent

Stable-ID upserts — reprocessing never creates duplicates.

No silent loss

Retries with backoff + a dead-letter queue you can replay.

Deterministic refs

Numeric refs are never guessed — parked until the target arrives.

Tombstones

Deletions preserve history; verify reconciles the graph.

RepoMindTech · Correctness
Pluggable memory — swap in one line

Pick your backend

memory

Offline, deterministic, zero AI deps — ideal for CI.

cognee

Local AI: Ollama + embedded kuzu graph, fully offline.

cognee_cloud

Hosted AI — just a key + URL, nothing to self-host.

Same typed structural graph across all three, so behavior and tests transfer directly.

Learning & Growth

RepoMindLearning
What we learned

Correctness first, then scale

  • Graph traversal beats vector RAG for explainable, cited answers
  • Idempotency + tombstones make ingestion safe to re-run anytime
  • Real repos are big — bounded ingestion & rate limits matter
  • One pluggable memory interface = swap local ↔ cloud in a line
